class CallExpr

Defined at line 2943 of file clang/include/clang/AST/Expr.h

CallExpr - Represents a function call (C99 6.5.2.2, C++ [expr.call]).

CallExpr itself represents a normal function call, e.g., "f(x, 2)",

while its subclasses may represent alternative syntax that (semantically)

results in a function call. For example, CXXOperatorCallExpr is

a subclass for overloaded operator calls that use operator syntax, e.g.,

"str1 + str2" to resolve to a function call.

Create a call expression.

Note that you can use CreateTemporary if you need a temporary call

expression on the stack.

Parameters

Fn The callee expression,
Args The argument array,
Ty The type of the call expression (which is *not* the return type in general),
VK The value kind of the call expression (lvalue, rvalue, ...),
RParenLoc The location of the right parenthesis in the call expression.
FPFeatures Floating-point features associated with the call,
MinNumArgs Specifies the minimum number of arguments. The actual number of arguments will be the greater of Args.size() and MinNumArgs. This is used in a few places to allocate enough storage for the default arguments.
UsesADL Specifies whether the callee was found through argument-dependent lookup.

void shrinkNumArgs (unsigned int NewNumArgs)

Reduce the number of arguments in this call expression. This is used for

example during error recovery to drop extra arguments. There is no way

to perform the opposite because: 1.) We don't track how much storage

we have for the argument array 2.) This would potentially require growing

the argument array, something we cannot support since the arguments are

stored in a trailing array.

bool parseCommentBlock (OutlineParserInterface * parser, Entry * curEntry, const QCString & comment, const QCString & fileName, int & lineNr, bool isBrief, bool isJavadocStyle, bool isInbody, Protection & prot, int & position, bool & newEntryNeeded, bool markdownEnabled, GuardedSectionStack * guards)

Invokes the comment block parser with the request to parse a

single comment block.

Parameters

parser [in] The language parse that invoked this function. The comment block parse may invoke ParserInterface::parsePrototype() in order to parse the argument of a @fn command.
curEntry [in] The Entry to which the comment block belongs. Any information (like documentation) that is found in the comment block will be stored in this entry.
comment [in] A string representing the actual comment block. Note that leading *'s are already stripped from the comment block.
fileName [in] The name of the file in which the comment is found. Mainly used for producing warnings.
lineNr [in,out] The line number at which the comment block was found. When the function returns it will be set to the last line parsed.
isBrief [in] TRUE iff this comment block represents a brief description.
isJavadocStyle [in] TRUE iff this comment block is in "Javadoc" style. This means that it starts as a brief description until the end of the sentences is found and then proceeds as a detailed description.
isInbody [in] TRUE iff this comment block is located in the body of a function.
prot [in,out] The protection level in which this comment block was found. Commands in the comment block may override this.
position [in,out] The character position within where the comment block starts. Typically used in case the comment block contains multiple structural commands.
newEntryNeeded [out] Boolean that is TRUE if the comment block parser finds that a the comment block finishes the entry and a new one needs to be started.
markdownEnabled [in] Indicates if markdown specific processing should be done.
guards [in] Tracks nested conditional sections (if,ifnot,..)
